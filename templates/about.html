About This Project

How the Lunar XRF Dashboard was built — tools, architecture, and design choices.

Tech Stack

Backend
  • Flask — Python web framework for routing and API endpoints
  • Pandas — data ingestion, schema parsing, statistical computations
  • NumPy — random data generation and numerical operations
  • Plotly — server-side figure generation (JSON payloads)
Frontend
  • Tailwind CSS — utility-first styling via CDN
  • Plotly.js — client-side chart rendering from JSON payloads
  • Vanilla JS — Fetch API for AJAX, DOM manipulation
  • Jinja2 — server-side template inheritance and rendering

Architecture

The app follows a classic server-rendered Flask architecture with AJAX-driven interactivity. All heavy computation happens on the backend — the browser only renders and plots pre-computed data.

• Data Layer — CSV files are uploaded or generated and read into Pandas DataFrames. A parse_schema function auto-detects lat/lon columns and numeric element fields.
• API Layer — Flask endpoints serve schema metadata (GET /api/schema), computed Plotly figures (POST /api/plots), and statistical summaries (POST /api/analysis).
• Rendering — Plotly figures are built server-side with Plotly Express, serialized to JSON, decoded from typed arrays, and rendered client-side with Plotly.react().
• Interactivity — Element selectors, bounding-box filters, and refresh buttons trigger new fetch() calls that re-compute everything server-side without a full page reload.

Data Generation

When no CSV is uploaded, the app generates a synthetic lunar XRF dataset with 2,000 samples and 12 geochemical elements:

{% for el in ['Al', 'Si', 'Mg', 'Fe', 'Ca', 'Na', 'K', 'Ti', 'Mn', 'P', 'Cr', 'Ni'] %} {{ el }}_wt_pct {% endfor %}

Each element follows a distinct statistical distribution influenced by rock type (Basalt, Anorthosite, Gabbro, Norite, Troctolite, Regolith). Elements are intentionally correlated — for example, Fe is anti-correlated with Al (plagioclase effect), Ti and Mn track Fe (mafic mineral association), and Ni correlates with Cr (compatible element pairing). Spatial coordinates are randomly distributed across lunar near-side regions.

Design Decisions

Server-side computation over client-side

The original analysis page calculated means, medians, and correlations in JavaScript by iterating over all rows. For large datasets this is slow and blocks the UI. Moving statistics to Pandas made the analysis page faster and enabled richer metrics (skewness, kurtosis, quartiles) at no extra cost.

Plotly JSON payloads instead of HTML

Rather than embedding Plotly figures as HTML or rendering them server-side as images, the API returns JSON figure descriptors. This allows the client to use Plotly.react() for smooth updates, preserves interactivity (hover, zoom, pan), and keeps the backend stateless between requests.

Dark theme with glass-morphism

The UI uses a dark space-inspired palette with translucent glass panels, a subtle radial gradient background, and neon green accents. Tailwind CSS handles responsive layout while a small custom CSS file adds backdrop blur effects and scrollbar styling.
